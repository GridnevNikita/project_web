# Общая информация
Проект представляет собой набор модулей для обработки финансовых данных, включающий функции для маскировки номеров карт и счетов, фильтрации и сортировки операций, а также форматирования дат.

## Структура проекта

### Модуль masks
Содержит функции для маскировки конфиденциальной информации:

* **get_mask_card_number** - маскирует номер карты (16 цифр)
* **get_mask_account** - маскирует номер счета (20 цифр)

Пример маскировки карты:
1234 56** **** 7890


Пример маскировки счета:
**7890


### Модуль widget
Содержит функции для обработки данных:

* **mask_account_card** - маскирует номера карт и счетов в строковом представлении
* **get_date** - преобразует дату из формата ISO в формат DD.MM.YYYY

### Модуль processing
Предоставляет инструменты для работы с данными операций:

* **filter_by_state** - фильтрация операций по состоянию (по умолчанию 'EXECUTED')
* **sort_by_date** - сортировка операций по дате (по убыванию по умолчанию)

## Tests
### Модуль test_masks 
* Тестирование правильности маскирования номера карты.
* Проверка работы функции на различных входных форматах номеров карт, включая граничные случаи и нестандартные длины номеров.
* Проверка, что функция корректно обрабатывает входные строки, где отсутствует номер карты.
* Тестирование правильности маскирования номера счета.
* Проверка работы функции с различными форматами и длинами номеров счетов.
* Проверка, что функция корректно обрабатывает входные данные, где номер счета меньше ожидаемой длины.

### Модуль test_widget
* Тесты для проверки, что функция корректно распознает и применяет нужный тип маскировки в зависимости от типа входных данных (карта или счет).
* Параметризованные тесты с разными типами карт и счетов для проверки универсальности функции.
* Тестирование функции на обработку некорректных входных данных и проверка ее устойчивости к ошибкам.
* Тестирование правильности преобразования даты.
* Проверка работы функции на различных входных форматах даты, включая граничные случаи и нестандартные строки с датами.
* Проверка, что функция корректно обрабатывает входные строки, где отсутствует дата.
### Модуль test_processing
* Тестирование фильтрации списка словарей по заданному статусу `state`.
* Проверка работы функции при отсутствии словарей с указанным статусом `state` в списке.
* Параметризация тестов для различных возможных значений статуса`state`.
* Тестирование сортировки списка словарей по датам в порядке убывания и возрастания.
* Проверка корректности сортировки при одинаковых датах.
* Тесты на работу функции с некорректными или нестандартными форматами дат.


## Примеры использования

```python
# Маскировка карты
mask_account_card("Maestro 1234567890123456") # Возвращает "Maestro 1234 56** **** 3456"

# Маскировка счета
mask_account_card("Счет 12345678901234567890") # Возвращает "Счет **7890"

# Фильтрация операций
operations = [
    {"id": 1, "state": "EXECUTED", "date": "2023-01-01"},
    {"id": 2, "state": "CANCELED", "date": "2023-01-02"}
]

filtered = filter_by_state(operations) # Возвращает операции только в состоянии EXECUTED

# Сортировка операций
sorted_ops = sort_by_date(operations) # Сортирует по дате по убыванию

# Форматирование даты
get_date("2023-01-01T12:00:00") # Возвращает "01.01.2023"
```
