# Проект по обработке банковских транзакций

## Общая информация
Проект представляет собой набор модулей для обработки финансовых данных, включая функции для маскировки номеров карт и счетов, фильтрации и сортировки операций, а также форматирования дат и интерактивного взаимодействия с пользователем через CLI.

## Структура проекта

### Модуль masks
Содержит функции для маскировки конфиденциальной информации:

- **get_mask_card_number** — маскирует номер карты (16 цифр)  
- **get_mask_account** — маскирует номер счета (20 цифр)  

Пример маскировки карты:  
```
1234 56** **** 7890
```

Пример маскировки счета:  
```
**7890
```

---

### Модуль widget
Функции для обработки данных и форматирования:

- **mask_account_card** — маскирует номера карт и счетов в строковом представлении  
- **get_date** — преобразует дату из формата ISO в формат DD.MM.YYYY  

---

### Модуль processing
Инструменты для фильтрации и сортировки операций:

- **filter_by_state** — фильтрация операций по состоянию (по умолчанию `EXECUTED`)  
- **sort_by_date** — сортировка операций по дате (по убыванию по умолчанию)  

---

### Модуль generators
Инструменты для генерации и обработки данных:

- **card_number_generator** — генератор номеров банковских карт в формате `XXXX XXXX XXXX XXXX`  
  - Генерирует последовательность номеров карт  
  - Форматирует номера в стандартный вид  
  - Поддерживает генерацию диапазонов номеров  
  - Проверяет корректность входных значений  

- **filter_by_currency** — фильтрация операций по валюте  
  - Работает с вложенными структурами данных  
  - Возвращает итератор с отфильтрованными операциями  

- **transaction_descriptions** — извлекает описания транзакций из списка операций  
  - Обрабатывает отсутствие описания  
  - Гарантирует наличие хотя бы базового описания  

---

### Модуль filters
Функции для расширенной фильтрации операций:

- **filter_by_description** — фильтрация транзакций по ключевому слову в описании  
- **count_by_category** — подсчет количества операций по заданным категориям описаний  

---

### Модуль data_loader
Функции для загрузки данных из файлов:

- **csv_transactions** — загрузка транзакций из CSV-файлов с разделителем `;` и кодировкой `utf-8`  
- **excel_transactions** — загрузка из Excel-файлов формата `.xlsx`  

Пример использования:

```python
from src.data_loader import csv_transactions, excel_transactions

transactions_csv = csv_transactions("../data/transactions.csv")
transactions_xlsx = excel_transactions("../data/transactions_excel.xlsx")
```

---

### Модуль main
Основной CLI-интерфейс программы:

- Позволяет пользователю выбрать источник данных (JSON, CSV, XLSX)  
- Загружает транзакции из выбранного файла  
- Предлагает интерактивно:  
  - Фильтрацию по статусу операции (`EXECUTED`, `CANCELED`, `PENDING`)  
  - Сортировку по дате (возрастание/убывание)  
  - Фильтрацию по валюте (например, рубли)  
  - Фильтрацию по ключевому слову в описании  
- Выводит итоговый список с отформатированной датой, описанием, маскированными номерами и суммой с валютой  
- Обрабатывает ошибки ввода и пустые данные с информированием пользователя

## Тесты

### Модуль test_masks
- Тестирование маскировки номеров карт и счетов на разных входных данных и граничных случаях  

### Модуль test_widget
- Тесты маскировки в зависимости от типа входных данных  
- Тестирование обработки некорректных данных  
- Проверка форматирования даты  

### Модуль test_processing
- Тестирование фильтрации по статусу операции  
- Тесты сортировки по дате, включая граничные случаи и ошибки формата  

### Модуль test_generators
- Тесты фильтрации по валюте, генерации карт и извлечения описаний транзакций  

### Модуль test_data_loader
- Проверка загрузки из CSV и Excel  
- Обработка ошибок чтения файлов и отсутствия файлов  
- Использование моков для имитации поведения функций  

### Модуль test_filters
- Тесты фильтрации по описанию (ключевому слову)  
- Тесты подсчёта количества операций по категориям

## Установка и запуск

1. Клонировать репозиторий  
2. Установить зависимости (например, через Poetry или pip)  
3. Запустить `main.py`  
4. Следовать подсказкам для выбора и фильтрации данных
